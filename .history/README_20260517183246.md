# ⚔️ NexusETL
 
**Automated League of Legends Esports ETL Pipeline**
 
NexusETL is a production-grade data engineering pipeline that extracts, transforms, and loads professional League of Legends esports data into a relational data warehouse, purpose-built for machine learning and predictive modeling.
 
---
 
## 🏗️ Architecture & Key Features
 
| Feature | Description |
|---|---|
| **DAG-Based Orchestration** | Managed by Prefect for strict sequential execution, task state tracking, and terminal-based observability |
| **Idempotency & State-Driven Execution** | Scripts query DB state before executing. If an API rate limit or network error drops a connection, cleanup scripts automatically identify missing records and backfill them on the next run without duplicating data (`ON CONFLICT DO UPDATE / DO NOTHING`) |
| **Data Leakage Prevention** | Feature engineering calculates rolling averages and historical standard deviations strictly up to the *previous* game, ensuring **zero look-ahead bias** for downstream ML models |
| **Relational Schema Automation** | Raw data is parsed and normalized across a PostgreSQL schema linking Matches, Players, Teams, Drafts, and Derived Features. Table creation is completely automated natively within the operational scripts |

## 🚀 Getting Started
 
### 1. Clone & Install
 
```bash
git clone https://github.com/yourusername/NexusETL.git
cd NexusETL
pip install -r requirements.txt
```
 
### 2. Configure the Environment
 
Create a `.env` file in the root directory using the provided `.env.example` and add your local PostgreSQL credentials:
 
```env
DB_NAME=RiftQuant
DB_USER=postgres
DB_PASS=your_password
DB_HOST=localhost
DB_PORT=5432
```
 
### 3. Run the Pipeline
 
Pass any valid tournament URL as a command-line parameter. The pipeline dynamically spins up the database structures on its first execution and cascades ingestion through all core modules:
 
```bash
python run_pipeline.py "https://gol.gg/tournament/tournament-matchlist/Worlds%202025%20Play-In/"
```

## ⚙️ Pipeline Execution Flow
 
The master orchestrator steps through execution in a strict sequential dependency order to maintain data integrity:
 
```
          [Tournament URL Input]
                    │
                    ▼
 1. Match Scraper             (matches_webscraper.py)
                    │
                    ▼
 2. Team Builder              (build_teams.py)              ──▶  Structural update: maps text strings to IDs
                    │
                    ▼
 3. Player Builder            (build_players.py)            ──▶  State check: skips unless new rosters needed
                    │
                    ▼
 4. Draft Scraper             (drafts_webscraper.py)        ──▶  Isolates maps with < 20 picks/bans logged
                    │
                    ▼
 5. Init Match Participants   (1_init_match_participants.py) ──▶  Seeds player/champion maps
                    │
                    ▼
 6. Update Advanced Metrics   (2_update_advanced_stats.py)  ──▶  Computes base game stats
                    │
                    ▼
 7. Init Derived Features     (3_init_derived_features.py)  ──▶  Creates modeling schema
                    │
                    ▼
 8. Generate Derived Features (4_compute_derived_features.py) ──▶  Vectors + final relational patch
                    │
                    ▼
            [ML-Ready Dataset]
```

## 🛡️ Key Implementation Details
 
- **Type Safety and Missing Values:** The pipeline uses Pandas' `Int64` nullable integer type to handle structural data anomalies (roster adjustments, temporarily missing team data) without casting failures.
- **Relational Self-Healing:** Dedicated backfill scripts automatically sync unmapped text objects to true relational foreign keys (`team_id`) across both primary tracking tables and derived feature databases, resolving out-of-order processing constraints during new tournament additions.
- **Efficient Processing Windows:** Computational complexity is optimized via Pandas vectorization. Rolling stats are computed over grouping windows using database indices optimized specifically for performance tracking.
