# ⚔️ NexusETL

**Automated League of Legends Esports ETL Pipeline**

NexusETL is a production-grade data engineering pipeline that extracts, transforms, and loads professional League of Legends esports data into a relational data warehouse, purpose-built for machine learning and predictive modeling.

---

## 🏗️ Architecture & Key Features

| Feature | Description |
|---|---|
| **DAG-Based Orchestration** | Managed by Prefect for strict sequential execution, task state tracking, and UI-based observability |
| **Idempotency & State-Driven Execution** | Scripts query DB state before executing. If an API rate limit drops a connection, cleanup scripts automatically backfill missing rows on the next run without duplicating data |
| **Data Leakage Prevention** | Feature engineering calculates rolling averages and historical standard deviations strictly up to the *previous* game, ensuring **zero look-ahead bias** for downstream ML models |
| **Relational Star Schema** | Raw JSON/HTML normalized into a PostgreSQL database linking Matches, Players, Teams, and Derived Features |

## 🚀 Getting Started

### 1. Clone & Install

```bash
git clone https://github.com/yourusername/NexusETL.git
cd NexusETL
pip install -r requirements.txt
```

### 2. Configure the Database

Create a `.env` file in the root directory using the provided `.env.example` and add your PostgreSQL credentials:

```env
DB_HOST=localhost
DB_PORT=5432
DB_NAME=nexusetl
DB_USER=your_user
DB_PASSWORD=your_password
```

Then run the schema setup scripts:

```bash
psql -U your_user -d nexusetl -f sql/schema.sql
```

### 3. Run the Pipeline

Pass any supported tournament URL as a parameter and the pipeline dynamically ingests data from any competitive region globally:

```bash
python run_pipeline.py "https://gol.gg/tournament/tournament-matchlist/Worlds%202025%20Main%20Event/"
```

## ⚙️ How It Works

```
Tournament URL
      │
      ▼
  [Extract]   ──▶  Scrape match metadata, draft order, player stats
      │
      ▼
 [Transform]  ──▶  Normalize JSON/HTML · Engineer lag features · Prevent data leakage
      │
      ▼
   [Load]     ──▶  Upsert into PostgreSQL star schema
      │
      ▼
  ML-Ready Dataset
```

---

## 🛡️ Design Principles

- **Idempotent by design:** safe to re-run at any point; state is always derived from the database, not runtime memory
- **Leakage-free features:** all rolling statistics are computed over a strictly historical window
- **Region-agnostic ingestion:** any `gol.gg` tournament URL works out of the box
